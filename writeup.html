Responsive Redesign

The goal of this assignment was to practive the workflow of redesigning a simple website. We took away the skills necessary to analyze and identify flaws in an existing interface, create low-fidelity and high-fidelity prototypes for various screen sizes, and build a responsive website based on those prototypes.

We first started off by picking a website to redesign. I picked a website for a massage studio. It was very outdated looking and had a poor layout. There were obvious flaws that could be corrected.

I analyzed the usability of the webpage using the criteria of:

  • Usability
  • Learnability
  • Memorability
  • Accessibility



The problems I found are as follows:

Usability:

  • Logo and pictures are cut off on small screen
  • Font is sometimes hard to read and small.
  • Drop down menu doesn't let you select options as it redirects every time you click it.
  • A few links that link to themselves.
  • Important info about the current state of the practice should be more obvious.

Learnability:

  • Links do not appear to be links and you sort of accidentally click them.
  • Text is dense.
  • About page should have more contact details.
  • Drop down menu doesn't show all the treatment options
  • It's hard to find her blog on the pages.

Memorability:

  • It might be hard to find contact details as they are sporadically placed around the page
  • It's hard to find her blog on specific procedures on the pages.

Accessibility:

  • There are no alt tags for the images.
  • There are a few empty headers that might confuse users.
  • Heading level is skipped from h1 to h3. The navigation menu contains redundant title text.
